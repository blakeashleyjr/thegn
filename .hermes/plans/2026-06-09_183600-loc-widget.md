# LOC Widget Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Embed `tokei` for fast code counting and display a lazily-updating LOC widget in the top right.

**Architecture:** We add `tokei` to the `watch` daemon, which already monitors the focused worktree for changes in the background. The daemon computes the LOC in a spawned thread (for extreme performance, avoiding blocking the diff paint) and caches it in SQLite. The `thegn stats` command reads the cached LOC for the currently focused worktree and exposes it alongside system stats. The `tabbar` plugin parses and renders it.

**Tech Stack:** Rust, `tokei`, SQLite, Zellij WASM plugin.

Add these methods to the `Db` impl (around line 226, after `put_diff_cache`):

```rust
    // --- LOC cache ---------------------------------------------------------
    pub fn get_loc_cache(&self, worktree: &str) -> Result<Option<usize>> {
        let r = self
            .conn
            .query_row(
                "SELECT loc FROM loc_cache WHERE worktree=?1",
                params![worktree],
                |r| r.get::<_, usize>(0),
            )
            .ok();
        Ok(r)
    }

    pub fn put_loc_cache(&self, worktree: &str, loc: usize) -> Result<()> {
        self.conn.execute(
            r#"INSERT INTO loc_cache(worktree,loc,fetched_at)
               VALUES(?1,?2,?3)
               ON CONFLICT(worktree) DO UPDATE SET loc=?2, fetched_at=?3"#,
            params![worktree, loc, util::now()],
        )?;
        Ok(())
    }
```

### Task 3: Compute LOC lazily in `watch` daemon

**Objective:** Compute LOC via `tokei` on a background thread when the worktree changes.

**Files:**

- Modify: `crates/thegn-cli/src/commands/watch.rs`

**Step 1: Write minimal implementation**

In `crates/thegn-cli/src/commands/watch.rs`, change `fn read_focus` to `pub fn read_focus`:

```rust

```

Add the `compute_and_cache_loc` function at the bottom of `watch.rs`:

```rust
/// Compute LOC using tokei on a background thread to avoid blocking the daemon.
fn compute_and_cache_loc(wt: String) {
    std::thread::spawn(move || {
        let mut languages = tokei::Languages::new();
        let config = tokei::Config::default();
        languages.get_statistics(&[&wt], &[], &config);
        let total_loc = languages.values().map(|l| l.code).sum::<usize>();

        if let Ok(db) = crate::db::Db::open() {
            let _ = db.put_loc_cache(&wt, total_loc);
        }
    });
}
```

In the `run` loop, call `compute_and_cache_loc` whenever `push_diff` is called. There are two places:

1. When focus changes:

```rust
                    watcher = make_watcher(tx.clone(), wt);
                    push_diff(&url, wt); // immediate paint for the new focus
                    compute_and_cache_loc(wt.to_string());
                    last_pr = 0; // force a PR refresh for the new worktree
```

2. After the debounce wake:

```rust
                // Coalesce a burst: keep draining until quiet for DEBOUNCE.
                while let Ok(()) = rx.recv_timeout(DEBOUNCE) {}
                if let Some(wt) = current.as_deref() {
                    push_diff(&url, wt);
                    compute_and_cache_loc(wt.to_string());
                }
```
