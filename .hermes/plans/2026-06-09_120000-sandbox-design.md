# Superzej Sandbox Architecture & Design Plan (Expanded)

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Expand the existing sandboxing architecture for Superzej workspaces by adding Network Port Management, Frontend UI Integration, Sandbox Resource Monitoring, and Garbage Collection.

**Architecture:**
The core abstraction (`sandbox::resolve` / `enter_argv`) is stable. We will expand it across four vectors:

1. **Network:** Add declarative port forwarding (`ports: ["8080:8080"]`) to `.superzej.toml` so developers can reach web servers running inside isolated NAT sandboxes.
2. **Frontend:** Expose the active sandbox backend to the Zellij WASM frontend (e.g. statusbar or sidebar) by recording the resolved backend into SQLite.
3. **Monitoring:** Add a `sandbox::stats()` API that fetches CPU/RAM utilization from the underlying runtime (via `podman stats` / `docker stats`) and surfaces it to the CLI/UI.
4. **Resilience (GC):** Add a garbage collection command to reap orphaned `superzej-*` containers that outlive their git worktrees (e.g., if a user manually runs `git worktree remove`).

**Tech Stack:** Rust, Podman/Docker CLI, SQLite (rusqlite), Zellij WASM Plugins.

// In sandbox.rs
#[derive(Debug, Default)]
pub struct SandboxStats {
    pub cpu: String,
    pub mem: String,
}

pub fn stats(spec: &SandboxSpec) -> Option<SandboxStats> {
    if !spec.backend.is_oci() {
        return None; // Host namespaces (bwrap) require cgroup scraping, skip for v1
    }
    let rt = spec.backend.binary();
    // format: CPUPerc|MemUsage (MemUsage usually outputs "50MB / 16GB", we split it)
    let argv = vec![rt, "stats", "--no-stream", "--format", "{{.CPUPerc}}|{{.MemUsage}}", &spec.name];

    let out = std::process::Command::new(argv[0]).args(&argv[1..]).output().ok()?;
    let stdout = String::from_utf8_lossy(&out.stdout);
    parse_sandbox_stats(stdout.trim())
}

fn parse_sandbox_stats(output: &str) -> Option<SandboxStats> {
    let parts: Vec<&str> = output.split('|').collect();
    if parts.len() != 2 { return None; }
    let mem = parts[1].split('/').next().unwrap_or(parts[1]).trim().to_string();
    Some(SandboxStats {
        cpu: parts[0].trim().to_string(),
        mem,
    })
}
```

// In sandbox.rs
pub fn identify_orphans(active_worktrees: &[String], container_names: &[String]) -> Vec<String> {
    let active_names: Vec<String> = active_worktrees.iter()
        .map(|w| container_name(w))
        .collect();

    container_names.iter()
        .filter(|c| c.starts_with("superzej-"))
        .filter(|c| !active_names.contains(c))
        .cloned()
        .collect()
}

pub fn run_gc(db_worktrees: &[String]) -> Result<(), String> {
    for backend in [Backend::Podman, Backend::Docker] {
        if !crate::util::have(backend.binary()) { continue; }
        let out = std::process::Command::new(backend.binary())
            .args(["ps", "-a", "--format", "{{.Names}}"])
            .output().map_err(|e| e.to_string())?;

        let stdout = String::from_utf8_lossy(&out.stdout);
        let containers: Vec<String> = stdout.lines().map(|s| s.to_string()).collect();

        let orphans = identify_orphans(db_worktrees, &containers);
        for orphan in orphans {
            let _ = std::process::Command::new(backend.binary())
                .args(["rm", "-f", &orphan])
                .output();
        }
    }
    Ok(())
}
```
