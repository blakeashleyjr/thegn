# Pinned Programs Phase 2 Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Implement floating/scratch pins, dynamic pin promotion, top-strip sizing, environment injection, and health monitoring/auto-restart for pinned programs.

**Architecture:** Extend the existing `[[pins]]` config and the `open_plan` resolution logic in `src/commands/pin.rs` to support floating layouts and environment variables. Health monitoring will tap into zellij pane/tab exit events tracked in the host event loop. Promotion logic requires interacting with zellij to capture the active pane's ID and command, then dynamically writing to `.superzej.toml` or injecting it into memory.

**Tech Stack:** Rust, zellij pipe API, termwiz
