# Thegn Config Experience & Polish Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Create a production-grade developer experience around `thegn` configuration by introducing JSON Schema generation, automating CLI overrides with macros to remove boilerplate, and adding a live-preview command palette action for theme accents.

**Architecture:**

1. **JSON Schema:** Add a `thegn config schema` subcommand that prints a JSON Schema generated from `thegn_core::config::Config` using the `schemars` crate. This allows users to get autocomplete in editors like VS Code and Neovim.
2. **Macro-Driven Overrides:** Replace the massive `match key` in `apply_override_str` with an automated traversal or macro so that developers don't have to manually wire up CLI dot-notation variables.
3. **Live Theme Previews:** Build a new `thegn menu` / palette action (`Theme: Preview`). It will cycle through a predefined list of accents and immediately rewrite `~/.config/thegn/config.toml` via AST manipulation, which in turn triggers the background daemon's live reload, rendering the new accent immediately without closing the palette.

**Tech Stack:** `schemars` (for JSON Schema generation), `toml_edit` (for theme patching), native palette (iocraft).

## Task 2: Refactor CLI Overrides (Macro/Reflection)

**Objective:** Automate `apply_override_str` so new config additions are automatically CLI-configurable without boilerplate.

**Files:**

- Modify: `crates/thegn-core/Cargo.toml` (Optional, if adding a proc-macro or reflection crate)
- Modify: `crates/thegn-core/src/config.rs`

**Step 1: Simplify via Serde/JSON traversal**
Since writing a custom procedural macro for dot-notation assignment can be heavy, the easiest way to map dynamic string paths like `"theme.accent"` into a strictly-typed `Config` struct is by doing a round-trip through `serde_json::Value` or `toml::Value`.

Replace the `match` inside `apply_override_str` with:

```rust
fn apply_override_str(cfg: &mut Config, key: &str, val: &str) -> Result<(), String> {
    // 1. Serialize current cfg to a dynamic Value tree
    let mut tree = serde_json::to_value(&cfg).map_err(|e| e.to_string())?;

    // 2. Traverse the dot-path
    let parts: Vec<&str> = key.split('.').collect();
    let mut current = &mut tree;
    for (i, part) in parts.iter().enumerate() {
        if i == parts.len() - 1 {
            // Leaf node: set the value. Try to infer bool/number, fallback to string
            if let Ok(b) = val.parse::<bool>() {
                current[*part] = serde_json::Value::Bool(b);
            } else if let Ok(n) = val.parse::<u64>() {
                current[*part] = serde_json::Value::Number(n.into());
            } else {
                current[*part] = serde_json::Value::String(val.to_string());
            }
        } else {
            // Drill down
            current = &mut current[*part];
            if current.is_null() {
                return Err(format!("Invalid path: {}", key));
            }
        }
    }

    // 3. Deserialize back into Config to validate Enums & Types
    let new_cfg: Config = serde_json::from_value(tree).map_err(|e| format!("Type error on {}: {}", key, e))?;
    *cfg = new_cfg;
    Ok(())
}
```

**Verification:** Run `cargo test -p thegn-core` and ensure all the CLI override tests still pass, meaning the dynamic JSON injection works perfectly.

**Step 2: Add Dispatch Logic**
In `crates/thegn-cli/src/palette/dispatch.rs`, intercept `theme-preview`:

```rust
    "theme-preview" => {
        // Because the palette currently acts as a one-shot menu,
        // to do a "live cycle", we either need to render a sub-menu
        // or just apply a random/next theme immediately.
        // Let's implement it as a sequential cycle for simplicity.

        let accents = ["#76eede", "#ff007f", "#00ff00", "#ffaa00", "#0088ff"];
        // Pick one randomly, or load config and find next
        let next_accent = accents[fastrand::usize(..accents.len())];

        // Write it safely using toml_edit to preserve comments
        if let Ok(mut doc) = std::fs::read_to_string(Config::path())
            .unwrap_or_default()
            .parse::<toml_edit::DocumentMut>()
        {
            doc["theme"]["accent"] = toml_edit::value(next_accent);
            let _ = std::fs::write(Config::path(), doc.to_string());
        }

        // thegn's file watcher will automatically pick this up in 500ms
        // and visually swap the theme behind the palette!
    }
```

Run `thegn`, press `Cmd-K`, select `Theme: Preview accents`. Watch the UI automatically change colors behind the palette popup without restarting.
